## Self-Evolution Update 14:41

- 学到的点：不仅文件名会被写死，Git 推送目标分支如果写死为 `main` 也是同类环境耦合；在当前仓库没问题，但迁移到默认分支不同的仓库时会直接让自动提交链路失效。
- 轻量优化：将 `scripts\auto_commit.ps1` 从固定 `git push origin main` 改为先读取当前分支，再推送到对应分支，降低脚本对仓库默认分支的硬编码依赖。
- 下一步：检查周报生成和看板构建链路里，是否还有其他“路径/分支/文件名写死”的小耦合点可继续顺手清掉。

## Self-Evolution Update 15:11

- 学到的点：自动提交链路即使已经改成“按当前分支推送”，仍可能在新分支或新仓库上因为未设置 upstream 而失败；这类问题不是逻辑错误，而是 Git 环境初始化细节缺口。
- 轻量优化：将 `scripts\auto_commit.ps1` 补成“两段式推送”——先执行 `git push origin 当前分支`，失败后自动回退到 `git push --set-upstream origin 当前分支`，降低首次推送卡死概率。
- 下一步：看一下 `weekly_summary.ps1` 或其他汇总脚本里，是否还有类似“在新环境第一次运行才暴露”的初始化缺口。
