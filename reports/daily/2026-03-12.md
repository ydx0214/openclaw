## Self-Evolution Update 23:49

- 学到的点：小红书 MCP 工具(xiaohongshu-mcp)虽然已经配置好了，但一直没有标准的工作流 playbook 来指导什么场景该用什么命令、完成后如何验证、以及如何与其他工作流（多媒体、浏览器）衔接。导致实际使用时容易走一步算一步，发布失败也无法追踪。

- 增量优化：新增 `playbooks/xiaohongshu-mcp-workflow.md`，把小红书 MCP 任务正式收口为三条主链路：
  1. 内容研究 & 趋势分析（搜索 -> 笔记分析 -> 证据包 -> 下游）
  2. 内容发布（素材准备 -> MCP发布 -> 验证 -> 下游）
  3. 信息流 & 发现（推荐流 -> 内容筛选 -> 分析/行动）
  
  同时明确前置条件（MCP服务器运行、登录验证）、执行命令、completion proof 和与多媒体/浏览器工作流的互联规则。

- 产出物：一本可复用的 `Xiaohongshu MCP Workflow` playbook，可直接用于小红书内容研究、竞品分析和笔记发布任务。

- 新方向探索：值得继续投入"其他 MCP 工具的标准工作流接入"这条能力线，尤其是当用户需要使用第三方 MCP 服务时，有标准 playbook 可以减少"功能能用但流程不规范"的返工。

- 规则/目标微调：以后凡是需要调用 MCP 工具的任务，默认先检查是否有对应的 workflow playbook；如有，按 playbook 执行；如无，可参考这个模板临时创建一个。

- 下一步：优先扩展小红书工作流到评论管理、复杂发布、多账号切换场景，或者探索其他 MCP 工具（如有）的标准工作流接入。
