- 新方向探索：值得持续投入“多媒体任务的结构化中间层”，尤其是从 brief 到 prompt pack、再到 iteration log 和已验收素材清单的完整接力。
- 规则/目标微调：当生图 / 视频 / 图生视频任务已锁定 brief，但还没进入实际生成时，默认先产出结构化 `Prompt Pack`，而不是直接给一大段混合 prompt，优先保证首轮结果可比较、可验收、可复用。

- 轻量优化：新增正式 playbooks/document-delivery-pack.md，把文档类任务的默认收口固化为 source of truth -> delivery artifact -> output check -> next-step handoff，并接入 PLAYBOOKS.md、playbooks/document-task-routing.md 与 self-evolution-log/roadmap.md。
- 新方向探索：值得继续投入“文档类任务的末端交付标准化”，尤其是 Markdown->Word、PDF/Word 转交付、报告/简历之外的一般 Office 交付场景。
- 规则/目标微调：当文档任务已经完成编辑、转换或排版，但下一步还涉及上传、发送、人工复核或归档时，默认先做 Document Delivery Pack，不以“文件生成了”作为真正完成标准。
- 下一步：优先把 document-delivery-pack 接到更多高频文档入口，尤其是 Markdown 转 Word、DOCX/PDF 双格式交付和浏览器上传前的文档收口。
